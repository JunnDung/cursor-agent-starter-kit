# Review Workflow

## Goal

Review changes for correctness, safety, maintainability, and test coverage.

## Steps

1. Identify changed files.
2. Understand intended behavior.
3. Check correctness.
4. Check edge cases.
5. Check security.
6. Check tests.
7. Separate blockers from suggestions.

## Output

```txt
Status:
Findings:
Tests to add:
Recommendation:
```
